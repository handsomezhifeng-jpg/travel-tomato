# 旅游番茄钟 - 产品设计书

## 一、产品概述

**产品名称**：旅游番茄钟（TravelTomato）

**核心理念**：将番茄钟工作法与虚拟旅行结合，用户每完成一段专注时间，就相当于"驾车旅行"到达一个真实城市。通过沉浸式的驾车画面和旅行成就感，激励用户保持专注。城市覆盖全球，到达目的地后自动作为下一次出发点，形成连续的环球旅行体验。

**目标用户**：需要专注工作/学习，同时希望获得趣味反馈的用户。

**运行平台**：Android / iOS 手机端，竖屏适配，使用 uni-app 跨平台框架开发。

---

## 二、核心流程

```
首次：选择出发点 → 选择目的城市(3选1) → 番茄钟计时 → 到达目的地 → 城市介绍+记录日志 → 目的地自动成为新出发点
后续：直接选择目的城市(3选1) → 番茄钟计时 → 到达目的地 → 城市介绍+记录日志 → 目的地自动成为新出发点
```

**连续旅行机制**：用户到达一个城市后，该城市自动成为下一次的出发城市，无需重新选择出发点。用户的旅行轨迹会形成一条连续的全球旅行路线（如：上海→杭州→黄山→武汉→长沙→...→曼谷→...→巴黎）。

**功能说明**：
- 出发点为用户当前所在城市（上次到达的目的地，自动继承）
- 根据出发点，从全球城市数据库筛选 200km 半径内的城市
- 随机推荐 3 个城市供用户选择
- 每个城市卡片展示：城市名、国家、真实距离(km)、预计专注时长、一句话简介
- 地图上以 200km 为半径画圆，标注出发点和 3 个候选城市
- 提供"换一批"按钮，重新随机 3 个城市
- 用户点击城市卡片即选中，进入番茄钟页面

**距离数据来源**：
- 使用预置的全球城市间真实距离数据（公路/铁路实际距离，非直线距离）
- 数据来源方案：
  1. **主方案**：预计算全球主要城市间真实距离，存入 `city-distances.json`
  2. **在线方案**：调用 OpenRouteService / Google Distance Matrix API 实时查询
  3. **降级方案**：当无真实距离数据时，使用 Haversine 公式计算球面距离 × 1.3 系数近似

**专注时长计算公式**：
```
专注时长(秒) = 城市间真实距离(km) ÷ 5(km/s)
```

示例：
| 距离   | 专注时长 |
|--------|----------|
| 50km   | 10秒     |
| 100km  | 20秒     |
| 150km  | 30秒     |
| 200km  | 40秒     |

> **注意**：按 5km/s 的速度，200km 最多只需 40 秒，实际使用中这个时间非常短。建议在实际开发中可将速度调整为更合理的值（如 0.1km/s，则 200km = 2000秒 ≈ 33分钟，接近标准番茄钟时长），或者保留 5km/s 作为趣味设定。**本设计书按需求原文 5km/s 编写**。

**跨洋/孤立城市处理**：
当用户到达沿海城市或岛屿城市，200km 内可能没有足够的陆地城市时：
- 若 200km 内城市不足 3 个，自动扩大搜索半径（200→400→600km），直到找到 3 个城市
- 跨海城市之间也可选择（如 日本福冈 → 韩国釜山），视为轮渡/飞行
- 界面上显示"附近城市较少，已扩大搜索范围"提示

**功能说明**：

1. **视频背景**
   - `kaiche.mp4` 作为背景全屏循环播放（`loop` + `autoplay` + `muted`）
   - 视频铺满整个页面，使用 `object-fit: cover`
   - 视频上方叠加半透明深色遮罩层，确保文字可读

2. **路线进度条**
   - 顶部显示：出发城市 → 目的城市
   - 中间有一个移动的圆点，表示当前位置
   - 圆点按照实时进度移动

3. **番茄钟计时器**
   - 居中大字体显示倒计时（MM:SS 格式）
   - 计时器样式：圆形背景 + 半透明毛玻璃效果
   - 字体使用等宽数字字体，防止数字跳动

4. **里程信息**
   - 显示已行驶公里数 / 总公里数
   - 进度条动态填充

5. **控制按钮**
   - 暂停/继续按钮
   - 暂停时视频也暂停，计时停止
   - 长按或双击可放弃本次番茄钟（需二次确认）

**视觉风格**：
- 整体沉浸感：驾车视频 + 深色半透明 UI
- 信息层使用 `backdrop-filter: blur(10px)` 毛玻璃效果
- 计时器数字使用渐变色（番茄红 → 橙色）
- 进度条使用流光动画效果

### 5.3 关键算法

**城市距离计算（Haversine 公式）**：

```typescript
function getDistance(
  coord1: [number, number],
  coord2: [number, number]
): number {
  const R = 6371; // 地球半径 km
  const [lon1, lat1] = coord1;
  const [lon2, lat2] = coord2;
  const dLat = toRad(lat2 - lat1);
  const dLon = toRad(lon2 - lon1);
  const a =
    Math.sin(dLat / 2) ** 2 +
    Math.cos(toRad(lat1)) *
    Math.cos(toRad(lat2)) *
    Math.sin(dLon / 2) ** 2;
  const c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a));
  return Math.round(R * c);
}
```

**200km 内城市筛选 + 随机3选（自动扩大半径）**：

```typescript
function getNearbyCities(origin: [number, number], allCities: CityData[]): CityData[] {
  let radius = 200;
  let nearby: CityData[] = [];

  // 自动扩大搜索半径，直到找到至少3个城市
  while (nearby.length < 3 && radius <= 1000) {
    nearby = allCities.filter(city => {
      const dist = getDistance(origin, city.coord);
      return dist > 20 && dist <= radius;
    });
    if (nearby.length < 3) radius += 200;
  }

  // 随机取3个
  const shuffled = nearby.sort(() => Math.random() - 0.5);
  return shuffled.slice(0, 3);
}
```

**真实距离获取（带缓存）**：

```typescript
async function getRealDistance(from: CityData, to: CityData): Promise<number> {
  const cacheKey = `${from.name}_${from.country}__${to.name}_${to.country}`;

  // 1. 查预置距离数据
  const preset = presetDistances[cacheKey];
  if (preset) return preset;

  // 2. 查本地缓存
  const cached = await distanceCache.get(cacheKey);
  if (cached) return cached;

  // 3. 调用 OpenRouteService API 查真实路线距离
  try {
    const dist = await fetchRouteDistance(from.coord, to.coord);
    await distanceCache.set(cacheKey, dist); // 缓存结果
    return dist;
  } catch {
    // 4. 降级：球面距离 × 1.3 系数近似公路距离
    return Math.round(getDistance(from.coord, to.coord) * 1.3);
  }
}
```

```sql

CREATE TABLE pomodoro_records (
  id            TEXT PRIMARY KEY,

-- 距离缓存表
```
